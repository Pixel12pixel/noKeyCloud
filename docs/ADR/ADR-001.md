# ADR-001: Choosing a Database
### Status: Accepted

## Context
The project aims to deliver a cloud file storage system that requires a robust and scalable database for storing information about users, files, permissions, and other operational data. Selecting the appropriate database technology is critical to ensuring performance, security, and maintainability.

### Requirements
1. **Scalability**: The database must support a growing number of users and increasing data volume without significant performance degradation.
2. **Security**: The database must provide strong data protection mechanisms, including encryption, access control, and regular backup support.
3. **Performance**: The database must execute queries efficiently, especially for large datasets.
4. **Manageability**: The database should be easy to manage and supported by high-quality documentation and community resources.
5. **Compatibility**: The database must be compatible with the technologies and tooling planned for this project.
6. **Relational Capabilities**: The database should support relational data models to manage complex relationships among users, files, and permissions.

## Decision
PostgreSQL will be used as the primary database for this project.

## Alternatives Considered
1. **MySQL**

   **Pros**
   - Highly popular and broadly supported by the community.
   - Good performance for simple query workloads.

   **Cons**
   - Fewer advanced capabilities compared to PostgreSQL.
   - Less flexible for handling complex data relationships.

2. **MongoDB**

   **Pros**
   - Flexible document-oriented data model suitable for selected use cases.
   - Good horizontal scalability.

   **Cons**
   - Limited native support for relational modeling, which can complicate management of complex relationships.
   - Less mature security controls compared to PostgreSQL, unsuitable for this project's requirements.

3. **SQLite**

   **Pros**
   - Lightweight and easy to deploy, suitable for small projects or prototypes.
   - No server configuration required, enabling quick setup.

   **Cons**
   - Limited scalability and performance for large datasets.
   - Insufficient advanced security and management features for a production-grade cloud storage platform.

## Rationale
1. **Scalability**: PostgreSQL provides strong vertical scalability (additional resources on a single node) and can be extended for horizontal scaling scenarios.
2. **Security**: PostgreSQL offers advanced security controls, including encryption support, role-based access control, and strong auditing and monitoring capabilities.
3. **Performance**: PostgreSQL is optimized for high performance through advanced indexing, query planning, and reliable handling of large datasets.
4. **Manageability**: PostgreSQL is supported by comprehensive documentation and a large user community, which simplifies administration and troubleshooting.
5. **Compatibility**: PostgreSQL integrates well with a wide range of programming languages and ecosystem tools used in modern backend systems.
6. **Relational Capabilities**: PostgreSQL is a fully relational database, making it well-suited to modeling and managing complex relationships that are central to a cloud file storage domain.

## Consequences

### Positive
- High performance and scalability to support growth in users and stored data.
- Advanced security features that improve protection of user data.
- Strong operational support through mature tooling, documentation, and community expertise.

### Negative
- PostgreSQL may require more effort to configure and administer than some simpler alternatives.
- At very large data scale, additional tools and architecture patterns may be required, increasing system complexity.

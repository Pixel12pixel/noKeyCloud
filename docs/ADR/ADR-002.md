# ADR-002: Choosing Authentication Method
### Status: Accepted

## Context
The project requires a secure and efficient authentication method to verify the identity of users accessing the cloud file storage system. The chosen authentication method must provide strong security measures to protect user credentials and prevent unauthorized access, while also ensuring a seamless user experience.

## Requirements
1. **Security**: The authentication method must provide robust security features, including protection against common attacks such as brute force, credential stuffing, and phishing.
2. **User Experience**: The authentication process should be user-friendly and not overly complex, ensuring that users can easily access their accounts without unnecessary friction.
3. **Scalability**: The authentication method should be able to handle a growing number of users without significant performance degradation.
4. **Compatibility**: The authentication method must be compatible with the technologies and frameworks used in the project.

## Decision
The chosen authentication method for this project is to implement srp (Secure Remote Password) protocol for password-based authentication.

## Alternatives Considered
1. **OAuth 2.0**

    **Pros**
    - Industry standard for authorization, providing robust security features.
    - Flexibility to be used for both authentication and authorization.
    
    **Cons**
    - Complexity in implementation, requiring additional infrastructure and configuration.
    - Overhead in terms of token management and validation, which may not be ideal for simple authentication needs.
2. **Password-based authentication with bcrypt hashing**

    **Pros**
    - Simple and widely understood method for authentication.
    - bcrypt provides strong hashing with built-in salting, making it resistant to brute-force attacks.
    
    **Cons**
    - Requires secure handling of passwords and proper implementation to avoid vulnerabilities.
    - May not provide the same level of security as more modern authentication methods, such as srp, which does not transmit passwords over the network.
3. **Passwordless authentication using email or SMS-based one-time codes**

    **Pros**
    - Eliminates the need for users to remember passwords, improving user experience.
    - Can provide strong security if implemented correctly, as it relies on possession of the email or phone number.
    
    **Cons**
    - May introduce additional complexity in implementation and user management.
    - Potential issues with delivery of one-time codes, such as delays or failures in email or SMS delivery.

## Rationale
1. **Security**: The srp protocol provides strong security by allowing users to authenticate without transmitting their passwords over the network, reducing the risk of credential theft. It also includes mechanisms to protect against common attacks such as brute force and replay attacks.
2. **User Experience**: While srp may require users to set up their accounts with a password, it provides a secure and straightforward authentication process once implemented, without the need for additional steps such as token management or one-time code delivery.
3. **Scalability**: The srp protocol is designed to be efficient and can handle a growing number of users without significant performance degradation, making it suitable for the expected user base of the cloud file storage system.
4. **Compatibility**: The srp protocol can be implemented in various programming languages and frameworks, making it compatible with the technologies planned for this project.

## Consequences
### Positive
- Strong security for user authentication, reducing the risk of unauthorized access and credential theft.
- A user-friendly authentication process that balances security with ease of use.
- Compatibility with a wide range of technologies and frameworks, allowing for flexible implementation options.

### Negative
- Implementation complexity may be higher than traditional password-based authentication, requiring careful design and testing to ensure security and functionality.
- Users will need to create and remember passwords, which may not be as convenient as passwordless authentication methods, although this is mitigated by the security benefits of srp.
